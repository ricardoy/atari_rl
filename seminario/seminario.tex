\section{Modelos de recomendação}

\frame{ 

  Encontramos duas formas de resolver o problema: 

  \begin{itemize}

  \item Construção de consultas baseadas no documento inicial.
  \item Cálculo de distância entre documentos e posterior
    agrupamento. (\textit{Clustering})

  \end{itemize}

}

\subsection{Construção de consultas}

\frame
{
  \frametitle{Construção de consultas}

  Dado um documento construímos uma consulta a ser realizada em um
  sistema de RI.

  Dos resultados obtidos, supomos que os $n$ primeiros documentos são
  relacionados ao documento original.

}

\frame
{
  \frametitle{Construção de consultas}

  Temos duas opções:
  
  \begin{itemize}
  \item \textbf{Extrair termos} que sejam considerados relevantes ao documento
    inicial e utilizá-los em uma consulta a um sistema de
    RI tradicional (\textit{mais à frente}).
  \item \textbf{Utilizar o próprio documento} como ponto de partida em um
    sistema baseado em CNG (\textit{Contextual Network Graphs})
  \end{itemize}
  
}

\subsection{Cálculo de distância e agrupamentos}

\frame 
{ 
  \frametitle{Cálculo de distância e agrupamentos} 

  Nesse modelo, serão construídos agrupamentos de documentos
  (\textit{clustering}) baseados em alguma métrica. 

  A partir de um documento inicial, serão escolhidos outros
  pertencentes ao mesmo agrupamento.

}

\frame
{
  \frametitle{\textit{Clustering}}

  Em um \textit{clustering}, temos dois problemas principais: 

  \begin{itemize}
  \item Cálculo de distância entre elementos.
  \item Metodologia de construção dos agrupamentos.
  \end{itemize}
  
}

\frame
{
  \frametitle{Distância}

  Existem basicamente dois tipos de distâncias entre documentos:

  \begin{itemize}
  \item Baseada em freqüência de termos.
  \item Baseada em semântica.
  \end{itemize}
  
}

\frame
{
  \frametitle{Distância baseada em freqüência}

  Para cada termo $t$ num documento $d$ é calculado um peso baseado na
  freqüência de $t$ em $d$ e na raridade de $t$ na
  coleção. Representamos cada documento como um vetor desses pesos e a
  distância entre dois documentos é a distância entre seus vetores.

}

\frame
{
  \frametitle{Distância semântica}

  É calculada por meio de navegação em taxonomias, ou busca em
  dicionários.

  Faz uso de bases como \textit{WordNet} ou \textit{Wikipedia}.

}

\frame
{
  \frametitle{\textit{WordNet}}

  A \textit{WordNet} é um dicionário em que substantivos, verbos,
  adjetivos e advérbios são organizados em conjuntos de sinônimos,
  cada um representando um conceito, que são interligados por meio de
  relações semânticas.
  
}

\frame
{
  \frametitle{Distância semântica}

  % figuras lúdicas ou exemplos!!!!!!!!!  transformar em função
  % ``matemática''. e.g. entrada - processamento - saída

  Usualmente, são utilizadas as seguintes métricas:
  
  \begin{itemize}
  \item Medida baseada em caminho: $f(length(c_1, c_2))$, onde
    $length$ é o número de nós no menor caminho entre $c_1$ e $c_2$

    % é uma função do número de arestas em uma taxonomia que separam
    % duas palavras (conceitos) $w_1$ e $w_2$.
    
  \item Medida baseada em conteúdo de informação: $g(max_{c \in S(c_1,
    c_2)}[-\log p(c)])$, onde $S(c_1, c_2)$ é o conceito que
    generaliza $c_1$ e $c_2$, e $p(c)$ é a probabilidade de encontrar
    o conceito $c$ no \textit{corpus}.
    
  \item Medida de sobreposição de texto: é uma medida do número de
    termos comuns entre as definições de dois conceitos.
    
  \end{itemize}
  
}

\frame
{
  \frametitle{Agrupamento em \textit{Clustering}}

  Existem dois tipos:

  \begin{itemize}
  \item \textit{Flat clustering} -- cria os \textit{clusters} sem uma
    estrutura explícita que relacione os \textit{clusters} entre si.

  \item \textit{Clustering} hierárquico -- cria uma hierarquia, uma
    estrutura mais informativa que o conjunto não estruturado obtido
    no \textit{flat clustering}. 
  \end{itemize}
  
}

\frame
{
  \frametitle{\textit{Clustering}}

  \begin{block}{Prós}
    \begin{itemize}
    \item É utilizado no Google News.

    \item Já foi utilizado no Data Quality.
     
    \end{itemize}
  \end{block}

  \begin{block}{Contras}
    \begin{itemize}
      
    \item Depende da escolha de uma métrica e do agrupamento, o que
      aumenta o número de variáveis para otimização do processo.

    \end{itemize}
  \end{block}

}

\frame
{
  \frametitle{Distância semântica}

  \begin{block}{Prós}
    \begin{itemize}
      
    \item Por usar relações semânticas entre conceitos, é capaz de
      trazer resultados mais relevantes.
      
    \end{itemize}
  \end{block}
  
  \begin{block}{Contras}
    \begin{itemize}
      
    \item Definir os termos para comparação se análise for feita sobre
      todo o artigo. Em caso de existência de \textit{tags}, é trivial.

      
    \item \textit{WordNet} só existe em inglês e não cobre assuntos
      muito específicos.
      
    \end{itemize}
  \end{block}

}

\frame
{
  \frametitle{LSA}

  Geralmente, uma coleção de documentos é tratada como sendo um espaço
  determinado por uma matriz Termos-por-Documentos, onde cada elemento
  corresponde ao peso de um termo em um documento. 

}

\frame
{
  \frametitle{LSA}

  Latent Semantic Analysis (LSA ou Análise Semântica Latente) é uma
  forma de decompor essa matriz Termos-por-Documentos, permitindo a
  redução da dimensionalidade desta, mas preservando o máximo possível
  de informação da matriz original.

}

\frame
{
  \frametitle{LSA}

  A decomposição obtida (normalmente através de \textit{Singular Value
    Decomposition}), consiste em 3 matrizes; a primeira define um
  espaço Termos-por-Conceitos; a segunda define um ``peso'' para cada
  conceito; e a terceira define um espaço Conceitos-por-Documentos.

  Uma forma de identificar documentos recomendados é, dado o documento
  original, determinar quais estão mais próximos a ele no espaço
  Conceitos-por-Documentos.

}

\frame
{
  \frametitle{LSA}

  \begin{block}{Prós}

    \begin{itemize}
    \item Diminui a dimensão do espaço a ser trabalhado
    \item Permite determinar similaridade entre documentos em termos de ``conceitos''
    \end{itemize}
    
  \end{block}
  
  \begin{block}{Contras}
    
    \begin{itemize}
    \item Matrizes envolvidas deixam de ser esparsas, e portanto consomem mais memória
    \item Cálculo consome muito processamento % O (n*n*m) , O (n*m*m)
    \item Difícil de implementar
    \end{itemize}
    
  \end{block}

}
